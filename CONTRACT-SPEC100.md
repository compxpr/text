# SPEC100 Rule for Text Contracts

## Purpose

SPEC100 means contract work must capture and check content purpose, audience, source notes, review state, validation checks, artifacts, integrations, and handoff boundaries.

## Required closure proof

Substantial contract changes must document:

- content metadata impact
- audience impact
- review impact
- artifact impact
- fixture impact
- validation commands run
- final line: `SPEC100 unresolved_gaps=0`

## Rule

Contract work must preserve content ID, path, purpose, audience, source notes, review status, validation checks, artifacts, limitations, and fallback status.
